▌1.1 行动/核心循环
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【Gloomhaven】
• 每回合选2张能力卡，用1张上半+1张下半
• 手牌=生命资源，耗尽=力竭退场
• 攻击修正牌堆替代骰子，减少运气方差
• ★ 核心张力：每次行动都在消耗生存资源

【Spirit Island】
• 快/慢行动系统：快速在入侵者前执行，慢速在其后
• 同时行动选择，减少等待
• ★ 核心张力：时间顺序决定策略，非对称精灵各有节奏

【Pandemic / Forbidden Island】
• 每人每回合有限行动数（通常4次）
• 多目标（灭火/治病/拿宝藏）竞争有限行动
• ★ 核心张力：有限行动 vs 多处危机

【共性】
• 行动有"重量感"（消耗有限资源或机会成本）
• 选择之间有真实取舍
• 核心循环简洁可解释（"每回合做X，然后Y发生"）


▌1.2 角色成长/发展
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【Gloomhaven】
• 经验值→升级→解锁新能力卡
• 个人任务→角色退休→解锁新职业
• 装备购买/增强，攻击牌堆个人化（祝福/诅咒）
• ★ 成长是渐进且有实感的

【Spirit Island】
• 每精灵独特成长轨：获得新力量卡、解锁内在能力
• 非对称成长路径，不同精灵发展曲线不同
• ★ 成长与身份深度绑定

【Call to Adventure】
• 三幕结构：Origin → Motivation → Destiny
• 经验/ Triumph / Tragedy 积累为 Destiny 分
• 道德选择（Corruption 轨）影响叙事与结局
• ★ 成长与叙事、道德选择结合

【共性】
• 有明确"变强"或"变化"的反馈
• 成长路径与身份/主题一致
• 可选：装备/道具层、能力解锁、多结局


▌1.3 叙事与玩家能动性
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【显性叙事 vs 涌现叙事】（业界共识）
• 显性叙事：游戏写好的剧情、事件文本
• 涌现叙事：玩家在游玩中创造的故事（"我那局做了……"）
• 优秀设计：机制简单到不碍事，但足够有张力，促使玩家做出有记忆点的选择

【Dead of Winter】
• Crossroads 卡：特定条件触发叙事选择
• 玩家决策影响故事走向
• ★ 故事从玩家选择中涌现

【Betrayal at House on the Hill】
• 探索生成随机房间，Haunt 触发后规则大变
• 50种 Haunt 剧本，每局不同故事
• ★ 每局都是全新故事

【Gloomhaven】
• Legacy：永久贴纸改变地图/卡牌，密封信封解锁
• 城市/道路事件卡，个人任务驱动
• ★ 世界因玩家行动而永久改变（战役向）

【共性】
• 选择有后果（短期或长期）
• 可选：条件触发叙事、多分支、永久改变
• 避免：纯线性"发生在我身上"的无选择剧情


▌1.4 合作/半合作张力
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【优秀合作设计】（Meeples Together / BGG 共识）
• 竞争性的输赢条件：一个主目标 + 多种失败方式，形成"到处救火"的紧迫感
• 有限沟通：限制信息共享，避免一人指挥全场
• 随机化创造真正决策：不是可解的谜题，而是每次不同的权衡
• 资源管理是核心：手牌/ token /骰子等，创造紧张感

【Dead of Winter】
• 主目标（殖民地生存）+ 秘密个人目标
• 部分目标需背叛；投票放逐
• ★ 合作中的猜疑与道德抉择

【Nemesis】
• 明确半合作：生存+个人目标，可能互相妨碍
• 多重威胁：异形、船损、他人目标、随机事件
• ★ 计划被打破的"高潮时刻"

【共性】
• 共同威胁 + 个人利益 = 何时合作、何时自利的张力
• 可选：秘密目标、投票/放逐、有限承诺


▌1.5 难度与可重玩性
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【Spirit Island】
• 对手（Adversaries）与场景（Scenarios）提供可变难度
• 恐惧作为另一种胜利方式（不只消灭入侵者）
• ★ 多路径胜利 + 可调难度

【Gloomhaven】
• 场景难度等级，战役解锁新内容
• 95个场景 + 职业解锁
• ★ 内容量支撑长期重玩

【共性】
• 难度可调（新手/老手、简单/困难）
• 多角色/多路径/多剧本增加重玩价值


▌1.6 配件与主题呈现
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【顶级游戏常见做法】
• 角色/派系有辨识度（立牌、meeple、插画统一）
• 主题与机制一致（如 Spirit Island 的恐惧、入侵阶段）
• 开箱体验与收纳影响第一印象

【注意】
• 重量与价格不是目标；简洁游戏也可成为经典（Catan, Ticket to Ride）
• 批判性审视提醒：避免为"像 Gloomhaven"而堆配件与规则

本报告以设计提案文件夹内两份主线为准：
• **v0.6 线**（历史版本/功德轮回_v0.6_完整设计总结）：半合作、修行阶段、劫难、众生、交易
• **v3.5 线**（功德轮回_v3.5_最终版规则书）：4 职业（农夫/商人/学者/僧侣）、皈依、发愿、Monte Carlo 平衡

▌2.1 核心机制（v0.6 设计）
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

• **行动**：每轮 5 阶段（协商→行动→修行升级→事件→劫难）；每人每轮 2 行动
• **成长**：4 阶段修行（凡夫→初果→菩萨→圆满），舍弃资源+骰子判定，职业路径不同
• **共同威胁**：劫难轨道（0–50），每轮+4，三灾阈值
• **NPC/目标**：20 张众生卡，渡化需合作，超时惩罚
• **个人目标**：12 张发愿卡（利他/平衡/自利），秘密选 1，无破坏性发愿
• **道德/负面**：三毒（贪嗔痴），影响升级与结局
• **互动**：协商阶段可交易（即时/布施/承诺/借贷），功德不可交易
• **胜利**：团队 5 条件全达成→个人计分→唯一佛果（最高分）

▌2.2 核心机制（v3.5 规则书）
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

• **4 职业**：农夫、商人、学者、僧侣（与 v0.6 的僧侣/贵族/商人/农夫不同）
• **发愿**：每职业 2 种发愿（简单/困难等），持续奖励+达成/失败结算
• **皈依选择**：两条道路都有活路，影响资源与策略
• **平衡**：50+ 迭代，6 配置测试，综合排名分；全皈依+平衡 11.3% 差距（★★★★★）
• **失败**：v3.5 注明 100% 失败原因为渡化不足

▌2.3 设计提案中的优势（自我定位与批判性审视）
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

• 主题独特：佛教经济学，市面上少见
• 道德选择核心：业力、戒律、发愿，不只是"怎么赢"还有"怎么活"
• 社会角色非对称：不同身份的人生与资源结构
• 东方美学与教育潜力
• 批判性审视强调：不盲目追赶 Gloomhaven；警惕 Feature Creep；学习 Catan/Ticket to Ride 的简洁与"易学难精"

【对比结论】
• 已有：清晰 5 阶段、2 行动/轮、资源种类丰富、协商与交易增加互动
• 缺口：行动本身没有"越用越少"的稀缺感（无手牌/精力类设计）；同时行动可增加节奏与策略层
• 与批判一致：不宜简单加"精力条"复制 Gloomhaven，而应让现有资源（功德/舍弃）更有重量

└────────────────────┴─────────────────────────────┴─────────────────────────────┘

【对比结论】
• 强项：修行阶段是鲜明长板——有代价、有随机、有职业差异、与佛教"舍弃"契合
• 缺口：无装备/道具、无"解锁新能力卡"式成长；若未来扩展，可考虑极简法器（1 件/人）且不增加规则量
• 建议：保持"减法"——不为了像 Gloomhaven 而加装备系统，除非能强主题化（如"一件法器，一种抉择"）


▌3.3 叙事与玩家能动性
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

┌────────────────────┬─────────────────────────────┬─────────────────────────────┐
│ 维度                │ 优秀RPG/合作桌游             │ 功德轮回（v0.6）            │
├────────────────────┼─────────────────────────────┼─────────────────────────────┤
│ 事件文本            │ Dead of Winter Crossroads；  │ 32 专属+20 共享事件卡，     │
│                    │ Betrayal Haunt 剧本          │ 有叙事空间                  │
├────────────────────┼─────────────────────────────┼─────────────────────────────┤
│ 选择影响后果        │ 条件触发、多分支、业报       │ 发愿达成/失败、三毒、渡化   │
│                    │                             │ 成功/超时皆有后果           │
├────────────────────┼─────────────────────────────┼─────────────────────────────┤
│ 早期选择影响后期    │ Gloomhaven Legacy；因果牌    │ 因果业报在历史文档中提议过， │
│                    │                             │ 当前规则未显式实现           │
├────────────────────┼─────────────────────────────┼─────────────────────────────┤
│ 涌现叙事            │ "我那局选了X导致Y"           │ 发愿+交易承诺+谁冲阶段 3   │
│                    │                             │ 可形成故事，依赖玩家讨论     │
└────────────────────┴─────────────────────────────┴─────────────────────────────┘

【对比结论】
• 已有：事件卡、发愿、三毒、渡化、修行抉择都能产生"我们这局发生了什么"的叙事
• 缺口：无显式"因果牌/历史碑刻"（前期行为在后期触发）；若加，建议极简（如 3–5 张因果事件，不铺新系统）
• 建议：优先在现有事件卡中加入"若你之前做过X，则本次Y"的 1–2 行分支，低成本增强叙事感


▌3.4 合作/半合作张力

├────────────────────┼─────────────────────────────┼─────────────────────────────┤
│ 共同威胁            │ Pandemic 病毒；Spirit Island │ ★ 劫难轨道+三灾阈值，       │
│                    │ 入侵者；DoW 道德/僵尸        │ 共同威胁明确                │
├────────────────────┼─────────────────────────────┼─────────────────────────────┤
│ 多种失败方式        │ 道德归零/回合尽/目标未达    │ ★ 劫难≥50/功德不足/渡化失败  │
│                    │                             │ /三毒过高等，多路径失败      │
├────────────────────┼─────────────────────────────┼─────────────────────────────┤

└────────────────────┴─────────────────────────────┴─────────────────────────────┘

【对比结论】
• 强项：共同威胁+多败条件+秘密发愿+果位竞争，半合作结构完整；交易与承诺增加信任/背叛张力
• 与经典半合作一致：团队门槛（5 条件）+ 个人排名（佛果），类似 DoW"主目标+个人目标"
• 建议：保持当前无破坏性发愿与无弹劾的设计，以利家庭/教育场景；弹劾可作为可选规则写在附录


▌3.5 难度与可重玩性
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【对比结论】
• 已有：难度档位、职业与发愿组合、v3.5 系统化测试
• 缺口：无"另一种团队胜利方式"（如纯恐惧胜利）；可选扩展可考虑"渡化数极高/劫难极低"的替代胜利
• 建议：先固化 1–2 个难度（标准/简单），再考虑 1 个"困难"变体（劫难+1/轮或条件更严）

▌4.2 与参考游戏的定位对比（一句话）
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

• Gloomhaven：战役、手牌=生命、装备与升级——功德轮回不做战役、不做手牌管理，成长用阶段+舍弃+骰子
• Spirit Island：非对称精灵、快慢行动、恐惧胜利——功德轮回非对称在职业与发愿，无快慢行动，单一团队胜利路径
• Dead of Winter：秘密目标、叛徒、放逐——功德轮回秘密发愿、无叛徒、无放逐，半合作更温和
• Pandemic：多危机、有限行动、简明规则——功德轮回多败条件与劫难类似；规则量大于 Pandemic，需注意教学曲线
• Root：派系完全异质——功德轮回职业差异大但共享同一 5 阶段流程，异质度低于 Root，利于上手

▌5.1 设计原则（来自批判性审视）
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

• 不把"Gloomhaven 有而我们没有"直接当缺陷；找准自身定位
• 警惕 Feature Creep：宁可做减法，不堆机制
• 目标：用最少规则创造目标体验；易学难精优于复杂难教
• 资源与道德重量优先于新增"精力/手牌"系统


▌5.2 高优先级改进（强化现有，少加新系统）
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【1】行动"重量感"（不新增精力条）
• 做法：在规则中显式写出"每轮 2 行动，不可存留"，并配 1–2 个范例（"若选修行+护法，则本轮回向/建寺不可做"）
• 可选：某 1 个"大行动"（如尝试阶段升级）消耗 2 行动，使回合内必须在"升级"与"两件小事"间二选一
• 风险控制：不引入新资源；仅澄清与少量规则微调

【2】因果/叙事轻量增强
• 做法：在现有 32 张专属事件中挑 3–5 张，增加"若你本局曾做过X（如布施≥3 次/曾违背承诺），则触发Y"
• 不单独做"因果牌堆"；不增加新阶段
• 风险控制：仅改事件卡文本与 1 行规则说明

【3】规则与流程可教性
• 做法：单页"一局流程"图（5 阶段+每阶段 1–2 句）；一页"胜利/失败条件"清单；玩家辅助卡统一术语（与规则书一致）
• 合并 v0.6 与 v3.5 的术语差异（如"功德"与"福/慧"对应关系），避免两套话术并存
• 风险控制：不改机制，只做文档与术语统一

【4】平衡与失败原因多样化
• 做法：v3.5 已指出"100% 失败=渡化不足"；在模拟或实测中微调渡化难度/劫难增速/众生出现率，使"功德不足""劫难爆表"等也占一定比例
• 目标：2–3 种主要失败原因并存，避免单一瓶颈
• 风险控制：数据驱动，小步调整


▌5.3 中优先级改进（可选或扩展）
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【5】同时行动（可选规则）
• 做法：在"行动阶段"中，某几轮改为"所有人秘密选 2 行动，同时亮出再结算"
• 收益：减少等待、增加猜测与配合
• 风险：需测试是否拖慢或混乱；建议标为"进阶规则"

【6】第二团队胜利路径（扩展）
• 做法：例如"若劫难≤5 且渡化≥8，可视为'净土'胜利"（与当前 5 条件二选一或替代其一）
• 增加策略维度与重玩感
• 风险：需重新平衡数值，建议作为扩展或变体

【7】1 件法器/圣物（极简装备）
• 做法：每人仅 1 件，从 4–6 张通用法器牌中选；效果为一句话被动或一次性主动
• 与佛教主题挂钩（如念珠、禅杖等），不铺装备栏与购买系统
• 风险：必须极简，否则违背"减法"原则


▌5.4 低优先级/不建议
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

• 精力系统：批判已指出与主题不符且增加负担；不采纳
• 完整装备/多件圣物：易导致 Feature Creep；不采纳除非做成独立扩展
• 朝廷弹劾/叛徒：与当前"无破坏性发愿"定位冲突；仅作可选附录
• Legacy/永久贴纸：与当前单局结构不符；不采纳


▌5.5 版本线统一建议
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

• 设计提案中并存 v0.6（僧侣/贵族/商人/农夫、修行阶段、劫难、交易）与 v3.5（农夫/商人/学者/僧侣、皈依、发愿、平衡数据）
• 建议：在文档中明确"基础版"与"扩展版"或"两条可选规则线"，并说明差异（职业集、是否有修行阶段、是否有皈依）
• 统一术语表（功德/福/慧/渡化/劫难等）避免读者混淆

▌6.1 总体结论
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

《功德轮回》当前设计（v0.6 线 + v3.5 线）在以下方面已对齐或接近优秀 RPG/合作桌游水准：

• 角色成长：修行阶段（舍弃+骰子+职业差异）有辨识度且与主题一致
• 合作/半合作：共同威胁、多败条件、秘密发愿、果位竞争、交易与承诺
• 主题与机制：佛教经济学与道德抉择深度嵌入
• 平衡与数据：v3.5 提供多配置测试与综合排名分

相对缺口主要在：

• 行动系统缺少"越用越少"的稀缺感（可通过澄清与"大行动"二选一轻量弥补）
• 叙事上缺少显式因果链（可通过少数事件卡分支低成本增强）
• 规则量与阶段数偏高，需依赖流程与术语整理降低学习成本
• 失败原因单一（渡化不足）可通过数值微调分散

整体不必追求"像 Gloomhaven 或 Spirit Island"；保持"佛教半合作+修行阶段+道德张力"的定位，在少加新系统的前提下做上述改进，更符合批判性审视中的简洁原则。


▌6.2 建议路线图（短期→中期）
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【短期（1–2 个迭代）】
1. 统一 v0.6 与 v3.5 的术语与规则表述，产出单页流程+胜负清单
2. 在 3–5 张事件卡中加入"若曾做X则Y"的因果分支
3. 明确"大行动"消耗 2 行动（或等价表述），并在规则中写清行动取舍范例
4. 根据 v3.5 数据微调渡化/劫难参数，使失败原因多样化

【中期（3–6 个迭代）】
5. 可选规则：同时行动阶段、1 件法器、困难难度
6. 扩展方向：第二团队胜利路径、更多众生/事件（保持牌池可控）
7. 真人盲测：3 组以上，收集"规则理解度""时长""道德抉择感""佛教代入感"
