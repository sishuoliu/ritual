▌2.2 基础版设计核心原则（研究总结）
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【原则1：核心机制必须坚实】
基础版建立的规则框架将支撑所有扩展内容。
→ 我们的核心：双层胜利、劫难轨道、众生渡化、发愿、布施

【原则2：15分钟学习规则】
成功的基础版应让新手在15分钟内理解规则并开始游戏。
→ 目标：规则书<6页，回合结构清晰，辅助卡完整

【原则3：共同威胁优先于个人冲突】
半合作游戏应以共同威胁驱动合作，而非玩家间对抗。
→ 我们的选择：劫难轨道是主要威胁，不设叛徒机制

【原则4：个人目标增加深度，不破坏合作】
个人目标应创造策略多样性，而非鼓励破坏团队。
→ 我们的选择：利他/平衡/自利型发愿，无冲突型

【原则5：预留扩展空间】
基础版应设计为可扩展的，核心规则不因扩展而改变。
→ 我们的设计：模块化扩展，逐步加入

▌4.1 回合结构优化
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【v0.7回合结构】
1. 协商阶段（2分钟）
2. 行动阶段（5分钟）
3. 事件阶段（3分钟）
4. 劫难阶段（1分钟）

【v0.8优化】
借鉴Nemesis的清晰回合结构，增加"收入阶段"

1. 【收入阶段】（新增，1分钟）
   • 商人自动+2财富
   • 已建寺庙：所有人+1功德
   → 目的：明确被动收益的触发时机

2. 【协商阶段】（2分钟）
   • 讨论策略
   • 进行布施
   → 简化：删除交易，仅保留布施

3. 【行动阶段】（5分钟）
   • 每人选择2个行动
   → 保持不变

4. 【事件阶段】（3分钟）
   • 翻众生卡（每轮2张）
   • 翻事件卡（每轮1张）
   → 简化：众生卡从"随机"改为"固定抽牌"

5. 【劫难阶段】（1分钟）
   • 劫难+3
   • 众生超时检查
   → 简化：删除中间灾难效果，仅保留失败条件

▌6.1 数学模型建立
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【游戏参数】
• 轮数：6轮
• 玩家数：4人
• 每人每轮行动：2次
• 总行动次数：6×4×2 = 48次

【资源流入分析】

起始资源总和：
• 功德：5+2+2+3 = 12
• 财富：2+10+8+3 = 23
• 影响：4+6+3+1 = 14

每轮固定收入：
• 商人贸易：+2财富/轮
• 寺庙（假设第3轮建成1座）：4轮×4人×1 = +16功德
• 寺庙（假设第5轮建成1座）：2轮×4人×1 = +8功德

行动产出（48次行动分配估计）：
• 修行×16：+32功德
• 劳作×12：+36财富
• 护法×8：-24财富，-16劫难
• 渡化×8：-约40财富/影响
• 建寺×2：-16财富，-8影响
• 专属×2：各种效果

【资源流出分析】

团队胜利需求：
• 渡化≥5众生：约-25财富（平均5财富/众生）
• 建寺≥2座：-16财富，-8影响
• 护法控制劫难：约-24财富

个人需求：
• 所有人功德≥10：总需≥40功德
• 发愿条件：各异

▌6.2 劫难轨道平衡验证
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【劫难增长】
• 每轮固定：+3
• 6轮总计：+18
• 众生超时（假设1个超时）：+3
• 负面事件（假设2张生效）：约+5
• 总增长：约+26

【劫难减少】
• 护法行动×8：-16
• 渡化众生×5：-10
• 正面事件（假设2张生效）：约-4
• 总减少：约-30

【劫难净值】
起始：0
终局：0 + 26 - 30 = -4 → 实际0（不能为负）

【结论】
目前平衡偏向玩家有利，略微调整：
• 调整：劫难每轮+3.5（取整为+4/-3交替）
  或：众生超时惩罚+4（原+3）

▌6.3 功德获取平衡验证
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【功德需求】
• 团队条件：所有人≥10 → 总需≥40
• 佛果级别（45+分）需功德约18+

【功德获取途径】
1. 起始：12（全队）
2. 修行行动：每次+2，估计16次 = +32
3. 渡化众生：每人每次+2，5次约 = +20（分给多人）
4. 寺庙产出：约+24（2座×4轮平均×4人×1/4参与）
5. 布施：约+6（18财布施/3）
6. 事件卡正面：约+4

总获取：12+32+5+6+4 = 约59（全队平均）

【人均功德】
59/4 = 约14.75功德

【结论】
• 平均功德超过10的门槛，但分布可能不均
• 调整：确保每个角色都有稳定的功德获取途径

▌6.4 角色平衡验证
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【僧侣】
起始：功德5/财富2/影响4
优势：功德起始高，讲经能力
劣势：财富最少
策略：专注修行+帮助他人
预期终局功德：18-22

【贵族】
起始：功德2/财富10/影响6
优势：资源丰富，赈灾效率高
劣势：功德起始低
策略：赈灾+建寺+后期修行
预期终局功德：14-18

【商人】
起始：功德2/财富8/影响3
优势：每轮+2财富，经济引擎
劣势：功德起始低，影响少
策略：积累财富→布施换功德
预期终局功德：14-18

【农夫】
起始：功德3/财富3/影响1
优势：布施效率×2
劣势：所有资源都少
策略：虔诚布施，每1财富=1功德
预期终局功德：16-20

【平衡结论】
• 僧侣功德最高但缺乏财务支持
• 贵族/商人资源丰富但需转化为功德
• 农夫资源最少但转化效率最高
• 需要合作才能各自达成目标

▌6.5 测试建议与数值微调
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【需要真人测试验证的假设】
1. 48次行动的典型分配
2. 布施行为的频率
3. 护法行动的使用时机
4. 众生超时的发生率

【如测试发现问题的调整方案】

若团队胜率过高（>70%）：
• 劫难每轮+4（原+3）
• 众生超时惩罚+4（原+3）
• 胜利条件：渡化≥6（原≥5）

若团队胜率过低（<40%）：
• 起始功德各+1
• 修行行动+3功德（原+2）
• 劳作行动+4财富（原+3）

若某角色获胜率过高：
• 检查该角色发愿是否过易
• 调整起始资源-1~-2

若某角色获胜率过低：
• 增强专属能力
• 调整起始资源+1~+2
