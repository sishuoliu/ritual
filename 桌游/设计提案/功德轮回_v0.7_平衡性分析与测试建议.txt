▌1.1 劫难轨道数学分析
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【基础数据】
• 游戏轮数：6轮
• 每轮劫难增长：+3
• 劫难失败阈值：20
• 团队胜利阈值：≤12

【自然劫难增长】
• 第1轮结束：3
• 第2轮结束：6
• 第3轮结束：9
• 第4轮结束：12
• 第5轮结束：15
• 第6轮结束：18

【分析】
• 不做任何护法，第6轮结束时劫难=18
• 低于失败阈值(20)，但高于胜利阈值(12)
• 需要至少降低6点劫难才能胜利

【众生超时惩罚】
• 每个众生超时：+3劫难
• 每轮翻开2张众生卡，2轮后超时
• 最多可能积累：12张众生 × 3 = 36劫难（极端情况）

【护法能力分析】
• 基础护法：消耗3财富，-2劫难
• 贵族赈灾：消耗5财富，-3劫难
• 渡化众生：-2劫难

【结论】
• 团队需要积极护法和渡化众生
• 预计需要3-5次护法行动才能胜利
• 难度中等偏高，符合设计目标


▌1.2 资源经济分析
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【4人起始资源总和】
• 功德：5+2+2+3 = 12
• 财富：2+10+8+3 = 23
• 影响：4+6+3+1 = 14

【6轮资源产出估算】

每轮2个行动点 × 4人 × 6轮 = 48个行动点

假设行动分配（估计）：
• 修行：12次（+24功德）
• 劳作：12次（+36财富）
• 护法：6次（-18财富，-12劫难）
• 渡化：8次（-约32财富，+16功德，-16劫难）
• 建寺：2次（-16财富，-8影响）
• 专属行动：8次

【资源平衡检查】

起始财富：23
+ 劳作产出：36
+ 商人贸易：12（6轮×2）
= 总财富：71

消耗：
- 护法：18
- 渡化：32
- 建寺：16
= 消耗财富：66

剩余财富：5（略有盈余）

【结论】
• 资源分配紧张但可行
• 需要团队合理规划
• 商人的贸易能力关键


▌1.3 功德积累分析
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【团队胜利条件】
• 所有人功德≥10

【各身份功德积累路径】

【僧侣】起始5功德
• 修行×4次：+8
• 渡化参与×2：+4
• 寺庙产出×4轮：+4（假设第3轮建成寺庙）
• 预计总功德：21

【贵族】起始2功德
• 修行×2次：+4
• 赈灾×3次：+3
• 寺庙产出×4轮：+4
• 预计总功德：13

【商人】起始2功德
• 修行×3次：+6
• 布施×2次：+4（假设布施12财富）
• 寺庙产出×4轮：+4
• 预计总功德：16

【农夫】起始3功德
• 修行×3次：+6
• 虔诚布施×3次：+6（3财富×2效率）
• 寺庙产出×4轮：+4
• 预计总功德：19

【结论】
• 所有身份都能达到10功德门槛
• 僧侣和农夫功德最高，符合佛教逻辑
• 商人和贵族需要更主动修行

▌2.1 测试目标
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【平衡性目标】

1. 团队胜率目标：40-60%
   • 太低：玩家受挫
   • 太高：缺乏挑战

2. 身份胜率平衡：每个身份获得佛果的概率接近25%±5%
   • 允许僧侣略高（符合主题）
   • 商人不应最低（避免被冷落）

3. 发愿完成率：利他型70%，平衡型60%，自利型80%
   • 利他型难但分高
   • 自利型易但分低

4. 游戏时长目标：45-60分钟


▌2.2 测试策略类型
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【策略A：全利他策略】
• 所有玩家选择利他型发愿
• 优先团队目标
• 积极布施和渡化

【策略B：混合策略】
• 2人利他型，1人平衡型，1人自利型
• 平衡个人和团队目标
• 最接近真实玩家行为

【策略C：全自利策略】
• 所有玩家选择自利型发愿
• 优先个人目标
• 最低限度合作

【策略D：最优化策略】
• AI计算最优行动
• 测试理论上限


▌2.3 测试场景
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

• 4名有经验玩家
• 充分沟通
• 合理分工

【场景2：新手团队】
• 4名新玩家
• 沟通有限
• 随机决策较多

【场景3：混合团队】
• 2名有经验 + 2名新手
• 部分指导

【场景4：独狼玩家】
• 1名自私玩家 + 3名合作玩家
• 测试游戏对自私行为的容忍度

▌3.3 潜在问题预警
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【问题1：僧侣过于强势】
• 风险：僧侣每局都获胜
• 原因：功德积累优势过大
• 对策：降低僧侣起始功德或讲经效果

【问题2：商人被边缘化】
• 风险：商人只负责供财，无存在感
• 原因：贸易能力被动
• 对策：增加商人专属发愿或成就分

【问题3：劫难失控太快】
• 风险：团队经常在第4-5轮失败
• 原因：众生超时惩罚累积
• 对策：降低劫难增长或超时惩罚

【问题4：游戏时间过长】
• 风险：超过60分钟
• 原因：协商阶段拖延
• 对策：使用计时器严格执行

▌4.1 如果团队胜率太低（<40%）
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【调整方案A：降低劫难压力】
• 劫难增长：3→2
• 失败阈值：20→25
• 超时惩罚：3→2

【调整方案B：增加资源产出】
• 劳作产出：3→4财富
• 修行产出：2→3功德
• 商人贸易：2→3财富/轮

【调整方案C：降低消耗】
• 建寺成本：8财富+4影响 → 6财富+3影响
• 护法成本：3财富 → 2财富
• 众生渡化成本：降低20%


▌4.2 如果团队胜率太高（>60%）
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【调整方案A：增加劫难压力】
• 劫难增长：3→4
• 胜利阈值：12→10
• 众生超时惩罚：3→4

【调整方案B：减少资源产出】
• 起始资源：各身份-10%
• 商人贸易：2→1财富/轮

【调整方案C：增加消耗】
• 建寺成本：+20%
• 众生渡化成本：+20%


▌4.3 如果僧侣胜率过高（>35%）
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【调整方案】
• 僧侣起始功德：5→4
• 讲经效果：他人+2功德 → 他人+1功德
• 僧侣修行效率：保持不变（不能削弱核心身份）


▌4.4 如果商人胜率过低（<18%）
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【调整方案】
• 商人起始功德：2→3
• 贸易加成：每轮+2财富 → +1财富+1功德
• 新增商人成就：资助最多额外+3分

▌6.2 模拟策略逻辑
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【利他策略AI】
```
优先级：
1. 如果劫难≥10，执行护法
2. 如果有众生即将超时，执行渡化
3. 如果建寺<2且资源足够，执行建寺
4. 如果他人功德<10，僧侣执行讲经
5. 执行修行
6. 如果有余力，布施给功德最低者
```

【自利策略AI】
```
优先级：
1. 如果自己功德<发愿需求，执行修行
2. 如果劫难≥15（危险），执行护法
3. 如果有众生即将超时且会导致失败，执行渡化
4. 执行劳作积累财富
5. 仅在必须时帮助团队
```

【混合策略AI】
```
50%概率使用利他策略
50%概率使用自利策略
（模拟真实玩家的不确定性）
```
