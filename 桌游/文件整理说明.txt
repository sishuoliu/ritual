三、上传 GitHub 注意
───────────────────────────────────────────────────────────────────────────────

• 仓库根目录已包含 README.md 与 .gitignore，可直接 git init / 添加远程 / push。
• .gitignore 已忽略：__pycache__、*.pyc、.vscode、tuning_results*.json、*.log 等。
• 中文路径在部分环境下可能显示异常，不影响克隆与使用；运行调优建议用根目录 run_merit_cycle_tuning.py。
• 大 PDF 若需保留在仓库，请勿在 .gitignore 中取消 *.pdf；若希望瘦身，可仅保留必要文献。


四、后续开发建议
───────────────────────────────────────────────────────────────────────────────

• 功德轮回：以 模拟测试/ 为代码主目录，设计文档集中在 设计提案/。
• 新规则或平衡参数：先改 模拟测试 中配置或事件表，跑 balance_tuning_v36.py 或 backtest_system，再同步更新 设计提案/功德轮回_规则说明书_v3.6.txt。
• 救赎之路：终版与规则在 终版/、05_游戏规则/；模拟器在 01_代码_模拟器/（若存在）。
