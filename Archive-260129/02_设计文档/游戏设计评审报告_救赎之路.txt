▌1.2 问题诊断
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【问题A：僧侣角色——致命设计缺陷】⚠️ 严重

原文描述："僧侣：无法直接获胜，但可影响其他玩家的功德分配"

问题分析：
• 这违反了"每个玩家都应有获胜机会"的基本平衡原则
• 没有理性玩家会自愿选择一个不能获胜的角色
• 如果强制分配，被分配到僧侣的玩家体验极差

参考案例：Root 的设计
Root 中每个派系（猫侯爵、鹰王朝、林地联盟、流浪者）都有完全不同的
获胜路径，但每个派系都有公平的获胜机会。流浪者虽然机制最独特，
但并非"辅助角色"。

修改建议：
僧侣应该有独特的获胜路径，例如：
• 通过"授德点数"累积获胜（授予他人功德也计入自己得分）
• 达成特殊宗教成就获胜
• 控制寺院影响力获胜

───────────────────────────────────────────────────────────────────────

【问题B：初始资源差距过大】⚠️ 中等

量化分析（假设粮食=1，铜钱=1.5，土地=3的价值比）：
• 富商：2×1 + 8×1.5 + 0×3 = 14 价值单位
• 地主：4×1 + 3×1.5 + 3×3 = 17.5 价值单位
• 官员：2×1 + 5×1.5 + 1×3 = 12.5 价值单位
• 农民：5×1 + 2×1.5 + 0×3 = 8 价值单位（+每轮2粮=20额外）
• 寡妇：2×1 + 2×1.5 + 0×3 = 5 价值单位
• 僧侣：0 价值单位

问题：
• 寡妇初始资源过低（仅5单位），回向加倍能力难以弥补
• 僧侣初始资源为0，且"不能拥有财产"使其无法参与资源循环
• 地主初始优势过大

───────────────────────────────────────────────────────────────────────

【问题C：特殊能力价值不均】⚠️ 中等

能力价值评估：
• 富商"交易"：被动能力，价值取决于其他玩家意愿
• 地主"产粮"：每块土地每轮+1粮食，3土地=每轮+3粮食，10轮=+30粮食（极强）
• 官员"皇宫"：取决于皇宫奖励设计，当前不明确
• 农民"劳作"：每轮+2粮食，10轮=+20粮食（强但不如地主）
• 寡妇"回向"：回向加倍，但需要先有功德才能回向
• 僧侣"授德"：只能影响他人，自己无法获益

问题：
• 地主的产出能力远超其他角色
• 寡妇的能力需要"先有功德才能回向"，但初始资源最少，形成悖论
• 僧侣能力与自身胜利无关

【修改方案B：僧侣角色全面重设计】⭐ 关键

新设计理念：僧侣是"功德银行家"，不拥有物质财产，但拥有独特的
"法力"资源，通过授予功德获得"授德点"，这是僧侣的专属得分。

新僧侣角色卡：
┌─────────────────────────────────────────────────────────┐
│  🧘 僧侣 THE MONK                                        │
├─────────────────────────────────────────────────────────┤
│  初始资源：                                             │
│  • 粮食：0（由信众供养）                                │
│  • 铜钱：0（不持有世俗财物）                            │
│  • 土地：0                                              │
│  • 法力：5（新资源，仅僧侣拥有）                        │
├─────────────────────────────────────────────────────────┤
│  特殊能力：                                             │
│  • 接受供养：当其他玩家向寺院捐献时，僧侣获得            │
│    等值的"法力"点                                       │
│  • 授予功德：消耗法力为其他玩家授予功德，               │
│    每授予1点功德，僧侣获得1点"授德点"                  │
│  • 法会主持：僧侣可发起法会，所有参与者共享功德，       │
│    僧侣获得参与人数×1的授德点                          │
├─────────────────────────────────────────────────────────┤
│  胜利条件：                                             │
│  • 授德点 × 2 = 最终得分                                │
│  • 或达成"涅槃"：法力归零 + 授德点≥15 = 直接获胜       │
└─────────────────────────────────────────────────────────┘

设计理由：
• 僧侣现在有独特的资源（法力）和得分方式（授德点）
• 僧侣的利益与其他玩家捆绑：玩家捐献→僧侣获法力→僧侣授功德→玩家获功德
• 这创造了真正的"共生关系"而非单向依赖
• 僧侣有自己的"解脱"路径：涅槃条件

───────────────────────────────────────────────────────────────────────

【修改方案C：寡妇角色增强】

问题：寡妇初始资源最少，能力需要先有功德，形成"穷者愈穷"困境

新寡妇角色卡：
┌─────────────────────────────────────────────────────────┐
│  👤 寡妇 THE WIDOW                                       │
├─────────────────────────────────────────────────────────┤
│  背景：失去丈夫的寡妇，在传统社会中处于边缘地位，        │
│  但佛教为她提供了通过虔诚获得救赎的路径。                │
├─────────────────────────────────────────────────────────┤
│  初始资源：                                             │
│  • 粮食：2                                              │
│  • 铜钱：2                                              │
│  • 土地：0                                              │
│  • 初始功德：2（独有！代表虔诚之心）                    │
├─────────────────────────────────────────────────────────┤
│  特殊能力：                                             │
│  • 虔诚之心：捐献时功德获取+1                           │
│  • 回向增幅：回向给他人时，双方各获得+1额外功德         │
│  • 哀悼祈福：每轮可免费向亡者回向（获得1功德）          │

设计理由：
• 给予初始功德解决了"需要先有功德才能回向"的悖论
• "哀悼祈福"提供稳定的功德收入（符合寡妇为亡夫祈福的主题）
• 限制（不能访问皇宫、不能买土地）平衡了增强的能力

▌2.1 资源转化链问题
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【问题D：转化率不明确】⚠️ 中等

原设计只说明：粮食 → 铜钱 → 功德

缺失信息：
• 粮食和铜钱的兑换比例是多少？
• 铜钱换功德的比例是多少？
• 土地如何进入这个循环？

修改建议：明确汇率表
┌─────────────────────────────────────────────────────────┐
│  基础汇率表（可被事件/角色能力修改）                     │
├─────────────────────────────────────────────────────────┤
│  粮食 → 铜钱：3粮食 = 2铜钱（在村庄市场）              │
│  铜钱 → 粮食：2铜钱 = 2粮食（反向兑换损失）            │
│  铜钱 → 功德：基础3铜钱 = 1功德（在寺院）              │
│  土地 → 粮食：每块土地每轮产出1粮食                    │
│  土地 → 铜钱：出售土地可获得5铜钱                      │
│  铜钱 → 土地：购买土地需要6铜钱                        │
└─────────────────────────────────────────────────────────┘

设计理由：
• 买卖差价创造了"交易成本"，防止无脑套利
• 功德价格（3铜钱=1功德）使功德真正"珍贵"
• 土地成为"投资品"：前期投资，后期回报

───────────────────────────────────────────────────────────────────────

▌2.2 功德定价机制问题
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【问题E：供需调整机制模糊】⚠️ 轻微

原设计："如果本轮多人想要同一服务，价格上涨"

问题：
• 涨多少？
• 谁来决定涨价？
• 僧侣玩家是否有定价权？

修改建议：引入"需求阶梯"机制
┌─────────────────────────────────────────────────────────┐
│  需求阶梯定价                                           │
├─────────────────────────────────────────────────────────┤
│  每种寺院服务在每轮有"供应量"限制：                     │
│  • 诵经超度：每轮最多2人                                │
│  • 法会祈福：无限制（但功德被稀释：总功德/参与人数）    │
│  • 立碑刻名：每轮仅1人                                  │
│  • 供养僧侣：每轮最多3人                                │
│  • 捐建大殿：整局游戏仅1次机会                          │
│                                                         │
│  竞争机制：                                             │
│  如果需求>供应，进入"竞标"：                           │
│  出价最高者获得服务，其他人保留资源                     │
└─────────────────────────────────────────────────────────┘

设计理由：
• 限制供应创造真正的稀缺性
• 竞标机制增加谈判和决策深度
• 法会的"稀释"机制创造了有趣的囚徒困境

───────────────────────────────────────────────────────────────────────

▌2.3 回向机制问题
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【问题F：50%回向加成可能过强】⚠️ 中等

原设计：
• 玩家A将功德回向给玩家B
• 玩家B获得转赠的功德
• 玩家A获得"回向功德"加成（转赠量的50%）
• 寡妇角色回向加成翻倍（100%）

数学分析：
假设玩家A有10功德，回向给B：
• B获得10功德
• A获得10×50%=5功德（作为回向奖励）
• 系统总功德从10变成15（凭空增加50%！）

如果是寡妇：
• B获得10功德
• 寡妇A获得10×100%=10功德
• 系统总功德从10变成20（翻倍！）

问题：
• 这会导致"功德通胀"
• 玩家可能互相回向套利
• 游戏后期功德数值膨胀失控

修改建议：
┌─────────────────────────────────────────────────────────┐
│  平衡后的回向机制                                       │
├─────────────────────────────────────────────────────────┤
│  方案1：固定奖励（推荐）                                │
│  • 回向任意数量功德，回向者获得固定+1功德               │
│  • 寡妇回向者获得固定+2功德                             │
│                                                         │
│  方案2：比例上限                                        │
│  • 回向加成25%，上限为3功德                             │
│  • 寡妇加成50%，上限为5功德                             │
│                                                         │
│  方案3：一次性加成                                      │
│  • 每轮只有第一次回向获得加成                           │
│  • 防止连续回向套利                                     │
└─────────────────────────────────────────────────────────┘

───────────────────────────────────────────────────────────────────────

▌2.4 "解脱"特殊胜利条件问题
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【问题G："解脱"条件几乎不可能达成】⚠️ 严重

原设计条件：
• 功德 ≥ 20
• 声望 = 0（完全匿名捐献）
• 剩余资源 = 0（彻底放下）

问题分析：
1. 功德≥20 需要大量捐献，但捐献过程中很难避免获得声望
   （"立碑刻名"服务给+3声望，"捐建大殿"给+5声望）
2. 如果有任何声望>0，就无法达成条件
3. 这使"解脱"成为理论上存在但实践上不可能的选项

修改建议：
┌─────────────────────────────────────────────────────────┐
│  重新设计的"解脱"胜利条件                              │
├─────────────────────────────────────────────────────────┤
│  选项1：放宽条件                                        │
│  • 功德 ≥ 15（降低门槛）                                │
│  • 声望 ≤ 3（允许少量声望）                             │
│  • 剩余资源 ≤ 2（允许基本生存资源）                     │
│                                                         │
│  选项2：添加"匿名捐献"选项                              │
│  • 玩家可选择"匿名捐献"：获得功德但不获得声望          │
│  • 匿名捐献的功德-1（作为"隐藏身份"的代价）            │
│                                                         │
│  选项3：渐进式解脱                                      │
│  • 游戏中途可"出家"转变为僧侣                          │
│  • 放弃所有资源，将一半资源转化为功德                   │
│  • 之后按僧侣规则行动                                   │
└─────────────────────────────────────────────────────────┘

▌3.1 事件卡问题诊断
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【问题H：马太效应——强者愈强】⚠️ 中等

原"江南大旱"事件：
"粮食歉收，所有玩家粮食-2。但功德最高的玩家不受影响（佛祖庇佑）。"

问题：
• 功德最高的玩家通常也资源最多
• 保护强者、惩罚弱者会加剧不平衡
• 弱势玩家追赶更困难

原"无常来袭"事件：
"声望最低的玩家寿命-2。"

问题：
• 声望最低者通常是弱势玩家
• 减少寿命=减少行动机会=更难追赶
• 形成负反馈循环

修改建议：
┌─────────────────────────────────────────────────────────┐
│  平衡后的事件卡设计原则                                 │
├─────────────────────────────────────────────────────────┤
│  1. 追赶机制：负面事件应给弱势玩家补偿                  │
│     例："江南大旱"：所有玩家粮食-2，                   │
│          但粮食最少的玩家可从寺院领取救济粮+1           │
│                                                         │
│  2. 惩罚转化：惩罚应转化为其他机会                      │
│     例："无常来袭"：声望最低者寿命-1（减少惩罚），     │
│          但获得"体悟无常"token，可在任意时刻             │
│          兑换为+2功德（将危机转化为机会）               │
│                                                         │
│  3. 避免累积惩罚：同一玩家不应连续受到惩罚              │
│     规则："保护状态"——刚受到负面事件的玩家             │
│          下一轮免疫负面事件                             │

▌4.1 固定轮数问题
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【问题I：10轮固定结束可能导致节奏单一】⚠️ 轻微

分析：
• 所有玩家同时结束，缺乏紧张感
• 玩家可以精确计算剩余轮数，优化最后冲刺
• 缺少"意外死亡"带来的紧迫感

修改建议：
┌─────────────────────────────────────────────────────────┐
│  动态结束机制                                           │
├─────────────────────────────────────────────────────────┤
│  方案1：差异化寿命                                      │
│  • 不同角色初始寿命不同                                 │
│  • 富商/地主：8轮（世俗享乐缩短寿命）                   │
│  • 僧侣/寡妇：12轮（清修延长寿命）                      │
│  • 事件可增减寿命                                       │
│                                                         │
│  方案2：不确定结束                                      │
│  • 第8轮起，每轮结束时掷骰                              │
│  • 1-2：游戏立即结束                                    │
│  • 3-6：继续下一轮                                      │
│  • 最多到第12轮强制结束                                 │
│                                                         │
│  方案3：触发式结束                                      │
│  • 当任一玩家功德达到15，触发"末日轮"                  │
│  • 所有玩家再进行最后1轮                                │
│  • 然后游戏结束                                         │
└─────────────────────────────────────────────────────────┘

───────────────────────────────────────────────────────────────────────

▌4.2 每轮行动数问题
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

问题："根据回合数调整"不明确

修改建议：
┌─────────────────────────────────────────────────────────┐

│  每轮每位玩家有2个行动点（AP）                          │
│                                                         │
│  行动消耗：                                             │
│  • 移动1格：0 AP（免费）                                │
│  • 移动2格：1 AP                                        │
│  • 交易：0 AP（但需要双方同意）                         │
│  • 购买：1 AP                                           │
│  • 捐献：1 AP                                           │
│  • 回向：1 AP                                           │
│  • 访问皇宫：2 AP（官员：1 AP）                         │
│                                                         │
│  修正：                                                 │
│  • 农民每轮+1 AP（勤劳）                                │
│  • 富商交易时可额外获得+1 AP（商业头脑）                │
│  • 游戏后半程（第6轮起）所有玩家+1 AP（紧迫感）         │
└─────────────────────────────────────────────────────────┘

▌优先级排序
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【高优先级 - 必须修改】

1. ❌ 僧侣角色重设计
   问题：无法获胜
   方案：引入"法力"资源和"授德点"得分系统

2. ⚠️ 回向机制平衡
   问题：50%/100%加成导致功德通胀
   方案：改为固定奖励（+1/+2）或设置上限

3. ⚠️ "解脱"条件调整
   问题：条件不可能达成
   方案：放宽条件或添加"匿名捐献"选项

【中优先级 - 建议修改】

4. ⚠️ 资源汇率明确化
   问题：转化率不清楚
   方案：制作明确的汇率表

5. ⚠️ 初始资源再平衡
   问题：差距过大
   方案：调整数值，拉近差距

6. ⚠️ 事件卡马太效应
   问题：强者愈强
   方案：添加追赶机制和保护状态

【低优先级 - 可选修改】

7. 功德定价供需机制细化
8. 行动点系统明确化
9. 动态游戏结束机制
10. 新增碑铭系统和寺院土地经济

【总体评价】

这是一个概念出色、主题深刻的桌游设计，成功将学术论文的核心概念
转化为游戏机制。但在角色平衡和数值设计方面存在需要修复的问题。

优点：
✓ 主题与机制的结合富有创意
✓ 资源转化链设计符合"功德经济"主题
✓ 回向机制捕捉了论文中"功德转让悖论"的精髓
✓ 角色多样性提供了重玩价值
✓ 事件卡增加了叙事深度

需改进：
✗ 僧侣角色的胜利路径必须重新设计
✗ 回向机制需要数值平衡
✗ "解脱"条件需要变得可行
✗ 资源汇率需要明确化

【下一步建议】

1. 实施高优先级修改
2. 进行纸上原型测试（至少3-5局）
3. 收集玩家反馈
4. 根据测试结果微调数值
5. 考虑制作简化版规则（15分钟版）
