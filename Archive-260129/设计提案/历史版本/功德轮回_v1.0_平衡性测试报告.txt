【测试方法】
采用 Monte Carlo 模拟方法，通过大量随机模拟来评估游戏平衡性。

【AI策略定义】
┌──────────────────────────────────────────────────────────────────────┐
│  策略      │  描述                                                   │
│  ─────────────────────────────────────────────────────────────────  │
│  智能型    │  根据局势动态决策，优先处理紧急事务                     │
│  平衡型    │  均衡发展，适度帮助团队                                 │
│  利他型    │  优先团队利益，积极护法和渡化                           │
│  自私型    │  只修行，不参与团队行动                                 │
│  混合型    │  4种策略各1个玩家                                       │
└──────────────────────────────────────────────────────────────────────┘

【模拟参数】
• 每种配置模拟 3000 局
• 4人游戏，6轮
• 胜利条件：劫难≤12，渡化≥6

▌2.2 结果分析
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【关键发现1：自私策略必败】
全自私型 0% 胜率，证明合作是必须的。
→ 设计意图达成 ✓

【关键发现2：平衡策略最优】
全平衡型 99.8% 胜率，远高于其他策略。
→ 平衡发展是最佳策略

【关键发现3：纯利他反而失败】
全利他型只有 1.7% 胜率。
原因：过度帮助导致资源不足，无法完成足够渡化。
→ 需要"先顾好自己"才能"帮助他人"

【关键发现4：混合策略可行】
混合型 75.4% 胜率，说明团队中有不同策略的玩家也能成功。
→ 游戏容错率高

【关键发现5：智能策略表现】
智能策略只有32.9%胜率，低于平衡策略。
原因：智能策略过于积极处理紧急事务，消耗过多资源。
→ AI策略需要优化，但也说明"过度反应"不是好策略

│  官员    │  利他型    │  0.0%      │  16.1      │  过度帮助          │
│  僧侣    │  自私型    │  89.3%     │  27.7      │  ★ 关键发现 ★     │
└──────────────────────────────────────────────────────────────────────┘

【重要发现：僧侣的逆转】
在混合策略中，僧侣采用"自私"策略（只修行）反而获得89.3%胜率！

原因分析：
1. 僧侣初始功德高(3)
2. 修行每次+2功德
3. 讲法时虽然-1给别人+3，但自己也+1
4. 当其他玩家忙于帮助团队时，僧侣专注修行积累了最多功德

这体现了一个深刻的游戏洞察：
★ 如果团队中有人愿意承担"利他"责任，
★ 那么专注"自度"的玩家反而可能获胜

这正好对应佛学中的一个争议：
"利他者为团队牺牲，但自度者反而成就最高"

▌5.1 当前版本问题
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【问题1：僧侣过弱（智能策略下）】
僧侣在同策略对战中只有0.2%胜率。
原因：讲法机制虽然团队收益高，但个人收益低。

【问题2：农夫过强】
农夫在智能策略下有52.6%胜率。
原因：劳作稳定积累财富，渡化效率高。

【问题3：利他策略太差】
全利他只有1.7%胜率，可能让选择帮助团队的玩家感到沮丧。

▌5.2 建议调整（可选）
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【调整1：加强僧侣讲法】
当前：消耗1功德，目标+3功德，自己+1功德
建议：消耗1功德，目标+2功德，自己+2功德
效果：增加僧侣个人收益

【调整2：削弱农夫】
当前：劳作无特殊加成（已移除+1）
建议：保持现状，但增加其他角色特色

【调整3：增加利他激励】
建议：帮助他人渡化时，协助者也获得1功德
效果：利他行为有个人收益

▌5.3 保持设计的理由
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【反对过度调整的理由】

当前设计有一个有趣的特性：
• 混合策略中，"自私"的僧侣反而获胜
• 这不是bug，而是feature

这创造了一个深刻的游戏体验：
• 如果大家都选择"帮助团队"，谁来"收割"功德？
• 如果大家都选择"积累功德"，团队会失败
• 这是一个真正的社会困境

保留这个设计可以让玩家体验到：
"有人必须牺牲，但牺牲者可能不会获胜"
这正是许多现实世界困境的缩影
