▌成功案例研究
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【Dead of Winter】（BGG评分7.8，经典半合作）
┌──────────────────────────────────────────────────────────────────────┐
│  机制：                                                              │
│  • 团队目标：确保僵尸殖民地生存                                      │
│  • 个人目标：每人有秘密目标（可能冲突）                              │
│  • 叛徒存在：部分目标需要叛变                                        │
│                                                                      │
│  胜利条件：                                                          │
│  • 团队目标达成 + 个人目标达成 = 个人获胜                            │
│  • 团队目标失败 = 所有人失败                                         │
│  • 团队成功但个人目标未达成 = 个人失败                               │
│                                                                      │
│  张力来源：                                                          │
│  • 不确定谁是叛徒                                                    │
│  • 个人目标可能需要损害团队利益                                      │
│  • 资源分配的争夺                                                    │
└──────────────────────────────────────────────────────────────────────┘

【Nemesis】（科幻生存）
┌──────────────────────────────────────────────────────────────────────┐
│  机制：                                                              │
│  • 团队目标：生存并逃离飞船                                          │
│  • 个人目标：隐藏目标，可能需要破坏飞船                              │
│  • 允许多人获胜或无人获胜                                            │
│                                                                      │
│  关键设计：                                                          │
│  • 个人目标可能互相冲突                                              │
│  • 某些目标需要他人失败                                              │
└──────────────────────────────────────────────────────────────────────┘

【Archipelago】（殖民地经营）
┌──────────────────────────────────────────────────────────────────────┐
│  机制：                                                              │
│  • 团队目标：防止土著暴动                                            │
│  • 个人目标：秘密目标（开发领土、控制资源）                          │
│  • 隐藏角色：分离主义者想要暴动发生                                  │
│                                                                      │
│  关键设计：                                                          │
│  • 团队失败（暴动）= 所有人失败                                      │
│  • 团队成功后，比较个人目标达成度                                    │
└──────────────────────────────────────────────────────────────────────┘


▌半合作游戏的核心特征
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

┌──────────────────────────────────────────────────────────────────────┐
│  1. 双层胜利条件：                                                   │
│     • 第一层：团队必须达成基本目标（合作）                           │
│     • 第二层：个人在团队成功基础上比较高低（竞争）                   │
│                                                                      │
│  2. 团队失败惩罚：                                                   │
│     • 如果团队失败，所有人都输（包括领先者）                         │
│     • 这迫使玩家必须合作                                             │
│                                                                      │
│  3. 资源张力：                                                       │
│     • 有限资源需要分配                                               │
│     • 玩家在"团队利益"和"个人利益"间抉择                            │
│                                                                      │
│  4. 隐藏信息：                                                       │
│     • 个人目标通常秘密                                               │
│     • 不确定他人是否在帮助团队                                       │
└──────────────────────────────────────────────────────────────────────┘

▌2.1 核心机制设计
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【双层胜利系统】

┌──────────────────────────────────────────────────────────────────────┐
│  第一层：团队胜利（门槛）                                            │
│                                                                      │
│  必须全部达成：                                                      │
│  1. 劫难≤12                                                          │
│  2. 渡化众生≥5                                                       │
│  3. 建寺≥2                                                           │
│  4. 所有玩家功德≥12                                                 │
│  5. 至少1人达到修行阶段3                                             │
│                                                                      │
│  → 若未达成：全员失败，无人获胜                                      │
│  → 若达成：进入第二层，比较个人成就                                  │
└──────────────────────────────────────────────────────────────────────┘

┌──────────────────────────────────────────────────────────────────────┐
│  第二层：个人成就（竞争）                                            │
│                                                                      │
│  在团队胜利基础上，根据个人分数评定果位：                            │
│                                                                      │
│  • 佛果（55+分）：唯一佛果，最高成就                                 │
│  • 菩萨果（45-54分）：1-2名玩家                                      │
│  • 阿罗汉果（35-44分）：2-3名玩家                                    │
│  • 初果（25-34分）：其余玩家                                         │
│  • 凡夫（<25分）：虽团队胜利，但个人修行不足                         │
│                                                                      │
│  ★ 关键设计：                                                       │
│  • 只有最高分者获得"佛果"（唯一胜者）                                │
│  • 其他人虽然"得救"，但果位较低                                      │
│  • 团队失败，所有人都是"凡夫"（无果位）                              │
└──────────────────────────────────────────────────────────────────────┘


▌2.2 为什么半合作更适合佛教游戏？
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【佛教教义的多层次】

┌──────────────────────────────────────────────────────────────────────┐
│  1. 基础层：慈悲救世（合作）                                         │
│     • 所有人必须参与"拯救众生"                                       │
│     • 如果世界毁灭，所有人都堕入恶道                                 │
│     • 体现"众生一体"的佛教观                                         │
│                                                                      │
│  2. 修行层：个人证果（竞争）                                         │
│     • 即使世界得救，个人修行有高低                                   │
│     • 阿罗汉vs菩萨：修行路径不同                                     │
│     • 体现"因人而异"的修行差别                                       │
│                                                                      │
│  3. 发愿层：自利vs利他（道德困境）                                   │
│     • 地藏宏愿：全力利他，牺牲个人果位                               │
│     • 阿罗汉道：自证涅槃，对团队帮助小                               │
│     • 体现"自了汉"vs"菩萨"的佛教争论                                │
└──────────────────────────────────────────────────────────────────────┘

【完美契合点】
• 合作：体现"慈悲普度"
• 竞争：体现"修行差异"
• 半合作：既慈悲又有个人修行追求

▌各模式优缺点
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【纯合作（v0.5）】
✓ 优点：
• 完全契合"普度众生"精神
• 无社交压力（无叛徒猜疑）
• 适合家庭游戏

✗ 缺点：
• 可能出现"主导玩家"问题（1人指挥所有人）
• 个人成就感较弱
• 缺乏半合作的社交推理张力

【半合作（v0.6）】
✓ 优点：
• 保留合作的慈悲精神
• 增加个人竞争动力
• 创造"我该牺牲多少？"的道德困境
• 隐藏发愿增加社交推理
• 更多重玩价值（猜测他人目标）

✗ 缺点：
• 规则稍复杂
• 可能有玩家过度自私
• 需要平衡合作/竞争比例

▌胜利条件（v0.6）
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【团队门槛】（必须全部达成）
1. 劫难≤12
2. 渡化众生≥5
3. 建寺≥2
4. 所有玩家功德≥12
5. 至少1人修行阶段≥3

→ 若未达成：全员失败，游戏结束
→ 若达成：进入个人计分

【个人计分】（仅在团队胜利后）

个人分数 = 功德×2 + 发愿分 + 修行阶段分 + 成就分 - 惩罚

【果位评定】
• 佛果（55+分）：**唯一佛果**，游戏胜者
• 菩萨果（45-54分）：证得菩萨，但非最高
• 阿罗汉果（35-44分）：自证涅槃
• 初果（25-34分）：初入圣道
• 凡夫（<25分）：虽得救，但修行不足

【关键规则】
• 只有最高分者获得"佛果"称号
• 并列最高分：共同获得佛果
• 其他人也"获救"了，但果位较低
• 团队失败，所有人都无果位


▌发愿系统更新（v0.6）
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【发愿类型调整】

1. 【利他型发愿】（对团队有利）
   • 奖励分数：中等（18-25分）
   • 对团队贡献：劫难-8~-12
   • 示例：地藏宏愿、护法檀越、布施长者
   • 佛教意义：菩萨道，普度众生

2. 【平衡型发愿】（中庸）
   • 奖励分数：中等（15-20分）
   • 对团队贡献：中等
   • 示例：菩萨道、功德商人
   • 佛教意义：福慧双修

3. 【自利型发愿】（对团队帮助小）
   • 奖励分数：低（10-15分）
   • 对团队贡献：几乎无
   • 示例：阿罗汉果、富甲天下
   • 佛教意义：自了汉

4. 【冲突型发愿】（可能伤害团队）★新增★
   • 奖励分数：高（25-30分）
   • 对团队贡献：负面
   • 示例：灭佛之主、恶霸盗匪
   • 佛教意义：恶道，但可忏悔
   • 特殊：需要谨慎使用，避免过度破坏


▌公开发愿机制（v0.6更新）
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【第4轮开始可公开发愿】
• 公开后：若达成+5分，若失败-10分
• 公开的好处：其他人可以协助你
• 不公开：保持神秘，但无协助

【发愿揭露】
• 第6轮，所有玩家必须揭示发愿类型（不公开具体卡）
• 宣布自己是"利他型"、"平衡型"还是"自利型"
• 增加信息，但仍保留悬念


▌协商与承诺机制
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【协商阶段的承诺】
• 玩家可以许诺："我这轮会护法"
• 但可以违背承诺（选择自利行动）
• 违背承诺：+2嗔，-2影响
• 被发现后：其他玩家可拒绝与你合作

【借贷系统】
• 玩家可以"借"资源给他人
• 承诺"下轮归还"
• 若不归还：+3贪，-3影响
• 增加社交互动和信任考验

▌方案A：提供两种模式
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【基础规则书】
• 纯合作模式（v0.5）
• 适合首次游戏、家庭聚会
• 规则简单，强调合作

【进阶规则书】
• 半合作模式（v0.6）
• 在基础规则上增加：
  - 冲突性发愿卡
  - 唯一佛果机制
  - 借贷与承诺系统
  - 可选：朝廷弹劾
• 适合策略玩家

【优点】
• 满足不同玩家需求
• 渐进学习曲线
• 同一套配件，两种玩法


▌方案B：默认半合作（推荐）
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【核心设计】
• 默认采用半合作模式（v0.6）
• 团队必须合作，个人可以竞争
• 但不加入"破坏性"发愿（如灭佛）
• 竞争仅体现在"谁修行更高"

【发愿设计】
• 12张发愿卡：
  - 4张利他型（对团队帮助大，个人分中等）
  - 6张平衡型（平衡）
  - 2张自利型（对团队帮助小，个人分低）
  - 0张冲突型（删除破坏性发愿）

【胜利评定】
• 团队必须达成5个条件
• 个人比较果位高低
• 最高分获得"佛果"称号
• 但所有人都"得救"了

【佛教意义】
• 就像佛陀的弟子中：
  - 舍利弗智慧第一
  - 目犍连神通第一
  - 阿难多闻第一
  - 虽然都是阿罗汉，但专长不同
• 游戏中也是：虽然都得救，但果位有高低

                          结论与建议
═══════════════════════════════════════════════════════════════════════════════

【我的推荐：方案B（半合作，无破坏性发愿）】

理由：
1. ✓ 保留合作的佛教精神（团队必须拯救众生）
2. ✓ 增加个人竞争动力（争取最高果位）
3. ✓ 创造道德困境（牺牲个人vs利他）
4. ✓ 避免破坏性行为（删除灭佛等冲突发愿）
5. ✓ 符合佛教"弟子有高低"的现实
6. ✓ 增加重玩价值（每次组合不同）

【修改要点】
从v0.5到v0.6：
• 胜利评定：从"所有人都胜利"改为"最高分获佛果"
• 发愿设计：保留利他/平衡/自利三类，删除冲突类
• 增加机制：借贷系统、承诺与违背
• 可选机制：朝廷弹劾（高级规则）

【不改变的内容】
• 修行阶段系统（4阶段）
• 三毒系统
• 众生卡系统
• 劫难轨道
• 事件卡系统
